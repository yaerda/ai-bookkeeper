## 1. 产品愿景与目标

### 1.1 愿景
打造一款**零门槛、全场景**的 AI 智能记账应用——用户只需说一句话、拍一张照片、甚至什么都不用做（自动捕获支付通知），即可完成记账。

### 1.2 目标用户
| 用户画像 | 描述 |
|-----------|------|
| 🎓 年轻白领 (22-35) | 日常消费频繁，希望轻松掌握资金流向 |
| 🏠 家庭用户 (25-45) | 需要管理家庭收支，关注月度预算 |
| 📱 移动优先用户 | 手机支付为主，习惯微信/支付宝 |

### 1.3 核心价值主张
> **"记账不该是负担——让 AI 替你记。"**

- ⚡ **极速记账**: 3 秒内完成一笔记录
- 🧠 **AI 理解**: 自然语言 → 结构化账单
- 📸 **拍照识别**: 小票/截图自动提取
- 🔔 **自动捕获**: 支付通知零操作记账
- 📴 **离线优先**: 无网络也能正常使用

### 1.4 竞品分析

| 维度 | 随手记 | 微信记账本 | Money Forward | **AI 智能记账 (本产品)** |
|------|--------|------------|---------------|--------------------------|
| 输入方式 | 手动表单 | 手动表单 | 银行 API 自动 | **多模态 (文字/语音/拍照/自动)** |
| AI 能力 | 弱 (分类建议) | 无 | 弱 (分类) | **强 (NLU + OCR + 自动提取)** |
| 离线支持 | ✅ | ❌ | ❌ | **✅ 离线优先架构** |
| 上手成本 | 中 | 低 | 高 | **极低 (自然语言)** |
| 自动记账 | ❌ | ❌ | ✅ (日本市场) | **✅ 支付通知监听** |
| 数据隐私 | 云端 | 微信生态 | 云端 | **本地优先 + 可选云同步** |

**差异化优势**: 国内首款将 LLM 自然语言理解 + OCR + 通知监听三合一的记账应用，且采用离线优先架构保护隐私。

## 3. User Stories & 验收标准

### Epic 1: 核心记账 (P0)

#### US-1.1 文字输入记账
> **作为**用户，**我希望**通过输入自然语言描述来快速记账，**以便**不需要填写复杂表单就能记录消费。

**验收标准 (AC):**
- [ ] AC1: 首页提供文字输入框，支持输入如 "午饭花了 35 元" "星巴克咖啡 28"
- [ ] AC2: 输入提交后，AI 自动提取金额、分类、商户名称、时间
- [ ] AC3: 提取结果展示给用户确认，用户可修改后保存
- [ ] AC4: 如 AI 无法提取金额，提示用户手动补充
- [ ] AC5: 保存成功后显示 Toast 提示，输入框清空
- [ ] AC6: 离线状态下使用预设规则提取，联网后可重新用 AI 解析

#### US-1.2 手动表单记账
> **作为**用户，**我希望**通过传统表单手动输入账单，**以便**在 AI 识别不准确时有备选方案。

**验收标准 (AC):**
- [ ] AC1: 提供表单页面，包含：金额、类型(收入/支出)、分类、备注、日期时间
- [ ] AC2: 金额为必填项，其余可选（分类默认"其他"，日期默认当前）
- [ ] AC3: 分类支持二级分类选择（如 餐饮 > 午餐）
- [ ] AC4: 保存后返回首页，账单列表自动刷新

#### US-1.3 账单列表与详情
> **作为**用户，**我希望**查看我的记账历史记录，**以便**了解自己的消费情况。

**验收标准 (AC):**
- [ ] AC1: 首页展示按日期分组的账单列表（今天、昨天、更早）
- [ ] AC2: 每条记录显示：分类图标、描述、金额、时间
- [ ] AC3: 每日小计显示当日支出/收入合计
- [ ] AC4: 点击记录进入详情页，可编辑或删除
- [ ] AC5: 支持下拉刷新、上拉加载更多
- [ ] AC6: 列表为空时显示引导状态（空状态插画 + 引导文案）

#### US-1.4 分类体系
> **作为**用户，**我希望**系统预置常用消费分类，**以便**快速归类我的每一笔消费。

**验收标准 (AC):**
- [ ] AC1: 预置支出分类: 餐饮、交通、购物、娱乐、居住、医疗、教育、通讯、服饰、其他
- [ ] AC2: 预置收入分类: 工资、奖金、兼职、理财、红包、其他
- [ ] AC3: 每个分类配有图标和颜色
- [ ] AC4: 分类支持用户自定义添加（P2 阶段）

#### US-1.5 编辑与删除账单
> **作为**用户，**我希望**编辑或删除已有的账单记录，**以便**修正错误或移除重复记录。

**验收标准 (AC):**
- [ ] AC1: 详情页提供编辑按钮，点击后进入编辑模式
- [ ] AC2: 所有字段均可修改
- [ ] AC3: 删除需二次确认（对话框："确定删除这条记录？"）
- [ ] AC4: 删除/编辑后列表实时更新

### Epic 2: AI 智能提取 (P0)

#### US-2.1 自然语言理解
> **作为**用户，**我希望**AI 能理解我的口语化描述并自动提取关键信息，**以便**我只需要简单描述就能完成记账。

**验收标准 (AC):**
- [ ] AC1: 支持以下输入模式并正确提取：
  - "午饭 35" → 支出 ¥35, 分类: 餐饮
  - "打车去公司 花了15块" → 支出 ¥15, 分类: 交通
  - "收到工资 8000" → 收入 ¥8000, 分类: 工资
  - "星巴克拿铁 28元" → 支出 ¥28, 分类: 餐饮, 商户: 星巴克
  - "昨天买了件衣服 299" → 支出 ¥299, 分类: 服饰, 时间: 昨天
- [ ] AC2: 调用 Azure OpenAI API，使用 structured output (JSON) 返回提取结果
- [ ] AC3: API 调用超时（>5s）时回退到本地正则提取
- [ ] AC4: 提取结果包含置信度，低置信度字段高亮提示用户确认

#### US-2.2 提取结果确认与修正
> **作为**用户，**我希望**在 AI 提取后能预览并修正结果，**以便**确保记录的准确性。

**验收标准 (AC):**
- [ ] AC1: AI 提取后展示结构化预览卡片（金额、分类、商户、时间、备注）
- [ ] AC2: 每个字段可点击修改
- [ ] AC3: 一键确认保存，或取消放弃
- [ ] AC4: 修正后的数据用于未来模型优化参考（本地记录）

- [ ] AC2: 常用分类根据用户近 30 天的记账频率自动排序，取 Top 4

- [ ] AC4: 输入金额后一键保存，自动填充：分类=所选分类、时间=当前、类型=支出

- [ ] AC7: 用户可在设置中手动自定义通知栏快捷分类（覆盖自动排序）

#### US-10.4 通知栏记账成功反馈
> **作为**用户，**我希望**通过通知栏快捷记账后获得即时反馈，**以便**确认记录已保存且信息正确。

**验收标准 (AC):**
- [ ] AC1: 记账成功后发送一条临时通知（自动 3 秒后消失），内容示例："已记录：餐饮 ¥35.00"
- [ ] AC2: 临时通知包含「撤销」按钮，点击后删除刚刚保存的记录

- [ ] AC4: 如 AI 提取置信度 < 0.7，通知改为"已记录：餐饮 ¥35.00（建议确认）"，点击跳转详情页

## 4. 数据模型概要

### 4.1 核心实体

```
Transaction (账单记录)
├── id: Long (PK, auto-generate)
├── amount: Double (金额)
├── type: Enum (INCOME / EXPENSE)
├── categoryId: Long (FK → Category)
├── merchantName: String? (商户名)
├── note: String? (备注/原始输入)
├── date: LocalDateTime (消费时间)
├── createdAt: LocalDateTime (创建时间)
├── updatedAt: LocalDateTime (更新时间)
├── source: Enum (MANUAL / TEXT_AI / VOICE_AI / PHOTO_AI / AUTO_CAPTURE / NOTIFICATION_QUICK)
├── status: Enum (CONFIRMED / PENDING)
├── syncStatus: Enum (LOCAL / PENDING_SYNC / SYNCED)
└── aiConfidence: Float? (AI 提取置信度)

Category (分类)
├── id: Long (PK)
├── name: String (分类名)
├── icon: String (图标资源名)
├── color: String (颜色 HEX)
├── type: Enum (INCOME / EXPENSE)
├── parentId: Long? (父分类 ID，支持二级分类)
├── isSystem: Boolean (是否系统预置)
└── sortOrder: Int (排序)

Budget (预算)
├── id: Long (PK)
├── categoryId: Long? (null 表示总预算)
├── amount: Double (预算金额)
├── month: YearMonth (预算月份)
└── createdAt: LocalDateTime

MonthlyStats (月度统计 - 计算缓存)
├── month: YearMonth (PK)
├── totalIncome: Double
├── totalExpense: Double
└── categoryBreakdown: JSON String
```

### 5.2 核心用户流程

#### 流程 A: 文字智能记账 (核心路径)
```
首页 → 输入 "午饭35元" → AI 提取 → 预览卡片 (¥35/餐饮/今天)
  → [确认] → 保存成功 ✅ → 列表刷新
  → [修改] → 编辑字段 → 确认 → 保存成功 ✅
```

#### 流程 B: 语音记账
```
首页 → 长按麦克风 → 录音 "晚饭和朋友吃火锅花了一百二"
  → 语音转文字 → AI 提取 → 预览 (¥120/餐饮)
  → 确认 → 保存 ✅
```

#### 流程 C: 拍照记账
```
首页 → 点击相机 → 拍照/选图 → OCR 识别
  → AI 提取 → 预览 → 确认 → 保存 ✅
```

#### 流程 D: 自动捕获
```
[后台] 收到微信支付通知 → 解析金额/商户
  → 生成待确认记录 → 用户打开 App → 账单页"待确认"Tab
  → 逐条/批量确认 ✅
```

#### 流程 E: 通知栏快捷文字记账
```
任意界面 → 下拉通知栏 → 点击「文字记账」
  → 弹出快捷输入浮窗 → 输入 "奶茶15" → AI 提取 → 简洁预览 (¥15/餐饮)
  → [确认] → Toast "记账成功 ¥15" + 关闭浮窗 ✅
  → [详细编辑] → 跳转 App 完整编辑页
```

#### 流程 F: 通知栏快捷分类记账
```
任意界面 → 下拉通知栏 → 展开通知 → 点击「🍚餐饮」
  → 弹出金额输入键盘 → 输入 "28" → 确认
  → 保存 (¥28/餐饮/当前时间) → Toast "记账成功 ¥28" ✅
```

## 6. 非功能性需求

### 6.1 性能
| 指标 | 目标 |
|------|------|
| 冷启动时间 | < 2 秒 |
| 文字记账端到端 | < 3 秒 (含 AI 调用) |
| AI 离线回退 | < 500ms |
| 列表滚动帧率 | ≥ 60fps |
| 数据库查询 (月度) | < 200ms |

### 6.2 安全与隐私
- 所有数据默认存储在设备本地
- 仅 AI 提取时将文本发送到 Azure OpenAI (不发送图片原文件)
- 通知监听权限需用户明确授权 + 隐私说明
- 不收集任何个人身份信息
- 未来云同步使用端到端加密

### 6.3 可用性
- 支持 Android 8.0+ (API 26+)
- 适配手机竖屏 (暂不适配平板/横屏)
- 支持中文简体 (v1.0 仅中文)
- 无障碍: TalkBack 基础支持
- Material Design 3 设计规范

### 6.4 技术约束
- 架构: MVVM + Clean Architecture
- DI: Hilt
- 异步: Kotlin Coroutines + Flow
- 网络: Retrofit + OkHttp (Azure OpenAI 调用)
- 图片: Coil
- 图表: MPAndroidChart 或 Vico

## 9. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Azure OpenAI API 延迟/不可用 | AI 提取失败 | 本地正则引擎作为 fallback |
| 微信/支付宝通知格式变化 | 自动捕获失效 | 解析策略可配置更新、适配多版本 |
| 通知监听权限被拒 | 自动记账不可用 | 充分的权限说明 + 功能降级提示 |
| OCR 识别率不足 | 拍照记账体验差 | 支持手动修正 + 持续优化提示词 |
| 数据安全顾虑 | 用户不信任 | 强调本地优先、透明隐私政策 |
